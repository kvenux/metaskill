This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Project Is

MetaSkill is a GitHub Pages-hosted skill catalog for AI coding assistants (Claude Code, OpenCode, etc.). It:
1. Aggregates skills from multiple GitHub repos (anthropics/skills, JimLiu/baoyu-skills, ComposioHQ/awesome-claude-skills, cexll/myclaude)
2. Categorizes and security-scans them via a Python aggregator pipeline
3. Generates a static JSON API deployed to GitHub Pages
4. Ships a meta-skill (`skill/`) that lets users browse/install skills from the catalog

## Project Structure

```
aggregator/          # Python pipeline (runs in CI, not locally served)
  main.py            # Entry point: clone → parse → scan → categorize → generate
  config.py          # Source repos, 20 categories, security rules, 8 skill packs
  cloner.py          # git clone/pull source repos into aggregator/_repos/
  parser.py          # Extracts skills from SKILL.md frontmatter in each repo
  categorizer.py     # LLM-based + keyword-fallback categorization
  security_scanner.py # Pattern-based security scan (blocked/warning/clean)
  static_generator.py # Writes JSON files to site/api/
  llm_client.py      # OpenAI-compatible LLM client for categorization
  requirements.txt   # pyyaml, openai

site/                # Static output (committed or CI-generated, served by GitHub Pages)
  index.html         # Homepage
  api/               # JSON API endpoints (generated by aggregator)
  css/, js/          # Frontend assets

skill/               # The meta-skill itself (codeskill)
  SKILL.md           # Skill definition with frontmatter + AI instructions
  scripts/codeskill_cli.py  # CLI tool users run to browse/install skills
  references/API_REFERENCE.md

repos/               # (gitignored) local clones for development
```

## Running the Aggregator Locally

```bash
cd aggregator
pip install -r requirements.txt

# Set LLM env vars (required for categorization; skips LLM if unset)
export LLM_API_KEY=your_key
export LLM_API_BASE=https://api.openai.com/v1
export LLM_MODEL=gpt-4o

python main.py
# Output written to site/api/
```

The aggregator clones repos to `aggregator/_repos/` (gitignored). Subsequent runs do `git pull` instead of re-cloning.

## Deployment

**Deployed site:** `https://kvenux.github.io/metaskill`

**AI agent install instruction** (shown on homepage and in `setup` file):
```
Fetch the installation guide and follow it:
curl -s https://kvenux.github.io/metaskill/api/codeskill/setup | cat
```

To deploy, push to `main` — GitHub Actions does the rest automatically.

```bash
git add -A
git commit -m "your message"
git push origin main
```

### GitHub Actions workflow (`.github/workflows/`)

Triggers: push to `main`, daily cron at 02:00 UTC, manual `workflow_dispatch`.

Steps:
1. `cd aggregator && python main.py` — clones repos, generates `site/api/`
2. Uploads `site/` as a Pages artifact and deploys via `actions/deploy-pages`

**Secrets/vars required in the repo settings:**
| Name | Type | Purpose |
|---|---|---|
| `LLM_API_KEY` | Secret | LLM API key for categorization |
| `LLM_API_BASE` | Variable | LLM endpoint (default: `https://api-s.zwenooo.link/v1`) |
| `LLM_MODEL` | Variable | Model name (default: `gpt-5.4`) |

To trigger a manual deploy without a code change:
```bash
gh workflow run "Aggregate Skills & Deploy"
# or watch progress:
gh run list --workflow="Aggregate Skills & Deploy"
gh run watch <run-id>
```

## Key Data Flows

**Skill data shape** (produced by `parser.py`):
```json
{
  "id": "skill-id",
  "name": "Skill Name",
  "description": "...",
  "category": "development-tools",
  "tags": ["tag1"],
  "source": {"repo": "repo-id", "path": "skills/skill-id"},
  "security_status": "clean|warning|blocked",
  "security_warnings": []
}
```

**Static API endpoints** (generated in `site/api/`):
- `skills/index.json` — all skills list
- `skills/<id>.json` — individual skill detail
- `categories/index.json`, `categories/<id>.json`
- `tags/index.json`
- `packs/index.json`, `packs/<id>.json`
- `codeskill/` — meta-skill install files (SKILL.md, scripts, install-sh, install-ps1)

## Adding a New Skill Source

Edit `aggregator/config.py` → `SOURCE_REPOS` list. Each entry needs `id`, `url`, `skills_dirs`, and `raw_base`.

## Adding Categories or Packs

Edit `aggregator/config.py`:
- `CATEGORIES` dict for new categories (20 currently defined)
- `CATEGORY_KEYWORDS` dict for keyword-based fallback assignment
- `SKILL_PACKS` dict for curated packs
